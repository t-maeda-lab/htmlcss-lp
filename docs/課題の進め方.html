AI時代の開発アプローチ

現代では生成AIで一発でコードを生成できますが、学習目的では段階的なアプローチが重要です

悪い例：一発生成

"チーズアカデミーのサイトを完璧に作って"

それっぽいものが生成される
どこを修正すれば良いかわからない

良い例：段階的構築

1 大枠の設計（骨組み）
2 各セクションの実装
3 スタイリングの調整
4 細部の修正

推奨開発フロー

1

設計・構造を決める

まず骨組みだけ作る

<!-- まず骨組みだけ作る -->
<body>
    <header><!-- ヘッダー --></header>
    <main>
        <section class="hero"><!-- メインビジュアル --></section>
        <section class="about"><!-- About --></section>
        <section class="course"><!-- Course --></section>
        <section class="news"><!-- News --></section>
        <section class="access"><!-- Access --></section>
        <section class="contact"><!-- Contact --></section>
    </main>
    <footer><!-- フッター --></footer>
</body>
2

セクション毎に内容を実装

一つずつ中身を作っていく

<section class="hero">
    <div class="container">
        <h1>セカイを変えるチーズを作ろう</h1>
        <p>チーズ職人養成学校「チーズアカデミーTOKYO」</p>
    </div>
</section>
3

CSS でレイアウト実装

基本レイアウトから

/* 基本レイアウトから */
.container {
    max-width: 1200px;
    margin: 0 auto;
}

/* 横並びレイアウト */
.course-grid {
    display: flex;
    gap: 20px;
}
4

見た目の調整

最後に装飾

/* 最後に装飾 */
.hero {
    background-image: url('img/mainvisual.jpg');
    background-size: cover;
    text-align: center;
    padding: 100px 0;
}

問題解決のプロセス

「わからない」時の効果的な対処法

1. まず出力してみる

  • とりあえず動くものを作る
  • 完璧でなくてOK
  • エラーが出ても気にしない

2. 内容を確認する

  • ブラウザで表示を確認
  • 期待通りになっているか？
  • どの部分がおかしいか特定

3. 問題を特定して修正

  • 一つずつ問題を解決
  • 小さな変更を積み重ねる
  • 動作確認を繰り返す

}
